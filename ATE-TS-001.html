Cadastro

CAD-E-001
Erro de sistema no cadastro
Belvo / Cumbuca / navegador / redirecionamento · Frequência: Recorrente
Média ▾
O que o usuário vê
Tela de erro na Belvo ou Cumbuca durante o cadastro, o navegador retorna erro, ou o cliente não é redirecionado para o app do banco para concluir o fluxo.
Causa raiz / hipótese
Incompatibilidade de navegador · Cache corrompido · Instabilidade temporária na integração com Belvo ou Cumbuca · Falha de redirecionamento.
Passos
1🧪
TestePergunte em qual etapa o erro ocorreu: Belvo, Cumbuca, navegador ou redirecionamento para o app do banco.
2▶
AçãoOriente o cliente a fechar completamente o navegador ou app, reiniciar o aparelho e tentar novamente.
3▶
AçãoPeça que acesse pelo Google Chrome — outros navegadores podem ter incompatibilidades com o fluxo.
4✓
VerificarO cadastro avançou normalmente? Se sim, concluído. Se não, avance.
5📝
ObsSe persistir em múltiplos dispositivos, pode ser instabilidade da Belvo ou Cumbuca. Registre e acompanhe canais de status.
Resultado esperado
Cadastro avança normalmente pelo Chrome após reiniciar o aparelho.
Escalação
Quando escalar
Para quem
Canal / como
Erro persiste após todas as tratativas em múltiplos dispositivos

CAD-E-002
Código de acesso não recebido via WhatsApp
WhatsApp / Plataforma de envio · Frequência: Frequente
Média ▾
O que o usuário vê
Cliente informa que solicitou o código de acesso mas não recebeu nada no WhatsApp, mesmo aguardando.
Causa raiz / hipótese
Canal incorreto (SMS ao invés de WhatsApp) · Acesso pelo WhatsApp Web (código não vai para PC) · Número da Facio bloqueado pelo cliente.
Passos
1🧪
TesteConfirme se o cliente solicitou pela opção WhatsApp — e não SMS. São canais distintos.
2🧪
TesteVerifique se está acessando pelo celular. O código não é enviado para o WhatsApp Web (computador).
3▶
AçãoSe estiver no celular e não recebeu: oriente a reiniciar o aparelho e solicitar o código novamente.
4🧪
TesteSe ainda não chegar: verifique se o cliente bloqueou o número da Facio no WhatsApp.
5▶
AçãoSe bloqueado: oriente a desbloquear e solicitar o código novamente. Envie a imagem de apoio.
Resultado esperado
Cliente recebe o código via WhatsApp no celular e acessa o cadastro normalmente.
Escalação
Sem escalação prevista — orientar o cliente e registrar o atendimento.

Não conhece →solicite ao cliente outro número disponível
Resultado esperado

Retool · Frequência: Esporádico

Sem escalação prevista — orientar o cliente e registrar o atendimento.
CAD-E-005
Cliente não reconhece o produto contratado
Retool · Frequência: Esporádico
Alta ▾
O que o usuário vê
Cliente afirma não se lembrar ou não ter realizado a contratação do produto.
Causa raiz / hipótese
Esquecimento genuíno · Contratação por familiar com acesso ao dispositivo · Fraude — uso indevido de dados por terceiro.
Passos
1▶
AçãoTranquilize o cliente — explique que vai verificar junto com ele.
2▶
AçãoEnvie as informações principais: dados de cadastro, data da solicitação, vencimento, contrato e comprovante de depósito.
3✓
VerificarCliente reconheceu ao ver os dados? Se sim, retorne ao fluxo normal.
4🧪
TesteSe insistir que não reconhece: siga o artigo de Suspeita de Fraude. Solicite o extrato bancário do período.
5▶
AçãoVerifique no extrato: após receber o valor, ele foi enviado a terceiros? Esse é o principal indicador de fraude.
Atenção
⚠ Confirmada suspeita de fraude: escale imediatamente conforme o POP de Suspeita de Fraude. Não dê continuidade ao atendimento padrão.
Resultado esperado
Situação identificada como esquecimento (cliente reconhece) ou encaminhada como fraude para a equipe responsável.
Escalação

Canal / como
Cliente insiste em não reconhecer após ver todos os dados OU extrato indica envio a terceiros

Média ▾

Passos

2▶
AçãoOriente a tentar novamente mais tarde (aguardar pelo menos 30 minutos).
3▶
AçãoSugira atualizar o app do banco para a versão mais recente.
4📝
ObsSe persistir por mais de 24h, orientar o cliente a contatar o suporte do próprio banco.

Cliente autoriza o Pix Inteligente após aguardar e/ou atualizar o app do banco.
Escalação
Sem escalação prevista — orientar o cliente e registrar o atendimento.

PIX-E-006
Conta reprovada ou negada pela QI Tech
Retool · Frequência: Comum
Alta ▾
O que o usuário vê
Abertura de conta reprovada. No Retool aparece como Facio bank account rejected. Cliente não consegue utilizar os serviços.
Passos
1✓
VerificarConfirme o status no Retool: Facio bank account rejected.
2▶
AçãoInforme ao cliente que a conta não pôde ser aprovada e que não é possível utilizar os serviços no momento.
3▶
AçãoExplique que não há possibilidade de retentiva, mesmo após 3 meses. O processo é automático.
4▶
AçãoInforme que, com o tempo, se o perfil de crédito melhorar, o cliente poderá receber uma nova oferta sem Transferência Automática.
Atenção
⚠ Não prometa prazos específicos nem reavaliações manuais. O processo de reanálise é automático e baseado no perfil de crédito.
Resultado esperado
Cliente informado sobre a reprovação e perspectiva futura. Atendimento encerrado com orientação completa.
Escalação
Sem escalação prevista — orientar o cliente e registrar o atendimento.
